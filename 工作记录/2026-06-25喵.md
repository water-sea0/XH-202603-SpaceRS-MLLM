# 本周工作进展汇报

本周主要围绕赛题数据理解、模型调研以及实验环境准备三个方面开展工作，目前已经完成数据分析与统一方案设计，并开始进行候选模型部署验证。

## 一、数据集分析与任务重构

首先对赛题涉及的数据集进行了梳理和分析，包括：

* VRSBench
* LEVIR-MCI
* Levir-CC
* XLRS-lite
* XLRS-Caption
* XLRS-VG
* MME-RealWorld

在分析过程中发现，各数据集不仅存储格式不同，其对应的任务目标也存在较大差异。如果直接按照数据集名称组织实验，后续训练和评测会比较混乱。

因此尝试从模型能力而非数据集来源的角度重新聚类任务。

最终将所有数据归纳为以下几类：

### 单图 → 文本

* Image Captioning
* Visual Question Answering（VQA）
* Visual Grounding（Refer）

对应数据集：

* VRSBench
* XLRS Caption
* XLRS-lite
* XLRS-VG

### 双图 → 文本

* Change Captioning

对应数据集：

* LEVIR-MCI
* Levir-CC

### 双图 → 像素级输出

* Change Detection

对应数据集：

* LEVIR-MCI

通过这种方式，后续实验能够围绕模型能力展开，而不是围绕具体数据集展开，也更符合多任务统一训练的思路。

---

## 二、模型调研与候选模型筛选

在明确任务分类后，对近期遥感多模态模型以及通用视觉语言模型进行了调研。

将多篇论文中的实验结果统一整理到同一张表格中，从以下几个维度进行横向比较：

### （1）模型性能

重点参考：

* MME-RealWorld
* XLRS-Bench
* VRSBench

等公开基准结果。

### （2）模型开放性

重点关注：

* 是否开源
* 是否支持本地部署
* 是否支持LoRA微调

排除了 GPT-4V、Claude、Gemini 等闭源模型作为主要方案。

### （3）多任务能力

考察模型在：

* Caption
* VQA
* Grounding
* 高分辨率场景

等多个任务上的综合表现，而不仅关注单项指标。

### （4）工程可行性

包括：

* 社区生态
* 文档完善程度
* 训练框架支持情况
* 后续改造难度

### 调研结论

目前较有价值的候选模型主要集中在：

* Qwen2-VL
* InternVL
* LLaVA

几个系列。

其中：

Qwen2-VL在性能、开源程度、社区生态以及后续扩展能力方面表现较为均衡，因此决定优先进行部署验证。

## 三、统一数据格式设计与数据清洗

由于各数据集原始格式差异较大：

* JSON
* Arrow
* Conversation JSON
* 多选题格式

均有涉及。

为了支持后续多任务联合训练，对数据进行了统一清洗与重构。

整体方案以 LLaVA Conversation JSON 为基础格式。

在保持与主流训练框架兼容的前提下，进行了少量扩展。

### （1）单图与多图统一

将原有：

```json
"image": "xxx.jpg"
```

扩展为：

```json
"images": [...]
```

统一表示单图和双图任务。

其中：

* Caption、VQA、Grounding 为单图
* Change Captioning 为双图

避免后续针对不同任务维护不同格式。

### （2）增加任务前缀

参考 EarthDial 等工作的做法，在 human prompt 中增加任务标识：

* [caption]
* [vqa]
* [refer]
* [changecaption]

使不同任务能够在统一框架下进行训练和推理。

### （3）增加元数据字段

新增：

* task
* sensor
* resolution
* bands
* language
* source

等字段。

这些信息不直接参与训练，但能够支持后续：

* 数据统计
* 平衡采样
* 消融实验
* 数据追踪

等工作。

### 阶段性成果

目前已经完成各数据集格式分析与统一方案设计，为后续模型训练和实验管理提供统一接口。

## 四、Qwen2-VL部署验证

在完成模型调研和数据格式设计后，已开始尝试部署 Qwen2-VL 作为第一阶段验证模型。

目前主要工作包括：

* 环境配置
* 模型下载
* 推理流程搭建
* 数据格式适配

过程中遇到了一些环境与框架兼容问题，目前正在排查解决。

预计本周五前完成基础推理流程验证，实现：

统一数据格式 → 模型加载 → 图片输入 → 文本输出

的完整闭环。

后续将在此基础上开展 Caption、VQA 和 Grounding 等任务的初步测试。
