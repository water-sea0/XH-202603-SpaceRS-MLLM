**Key finding**: Qwen's OOM failures are **NOT caused by concurrent GPU usage**. Even when running solo with the full 32 GB available (25.8 GB base model + ~6 GB headroom), Qwen2.5-VL's dynamic resolution processor creates too many visual tokens from large XLRS remote sensing images. Each failed sample needs 7-8 GB additional memory on top of the 25.8 GB base footprint, exceeding the 32 GB total.

In contrast, EarthDial's fixed 448×448 resolution never triggers this issue — the same 100M-pixel images are safely resized before processing.

### EarthDial — Zero Failures

EarthDial processed all 889 samples without a single error, including the same XLRS images that crashed Qwen. Its fixed-resolution design is inherently more memory-stable.

## Key Findings

1. **EarthDial is the reliability winner** — 100% completion vs 84.7%. For production RS pipelines where every sample counts, this is decisive.

2. **EarthDial is faster** — 0.88s/sample solo vs Qwen's estimated ~0.85s/sample solo (2× faster than the 1.71s concurrent figure). Despite being a smaller model, EarthDial achieves comparable or better speed on RS data.

3. **VRAM efficiency** — EarthDial uses only 8.7 GB vs Qwen's 25.8 GB. This means EarthDial can run on consumer GPUs with substantial headroom for batch inference, while Qwen2.5-VL-7B barely fits on a 32 GB card and cannot handle large RS images.

4. **Dynamic resolution is a double-edged sword** — Qwen's adaptive tiling gives richer descriptions on successfully processed images, but the memory cost makes it unreliable for RS datasets which often contain very large source images.

5. **Output verbosity** — Qwen generates ~3.2× more tokens than EarthDial (44.6K vs 13.8K). For RS tasks like change detection and VQA where concise answers are preferred, EarthDial's brevity is an advantage.

## Qualitative Observations

- **Change detection**: EarthDial is precise ("no_change", "a building appears"). Qwen is more verbose but sometimes hallucinates details.
- **VQA**: Both models answer correctly. EarthDial is more direct; Qwen adds explanatory context.
- **Visual grounding**: EarthDial outputs bounding box coordinates reliably. Qwen does well on small images but OOMs on large ones.
- **Image captioning**: Qwen gives richer descriptions when it works; EarthDial's descriptions are functional but less detailed.

## Recommendations

- **For RS production**: EarthDial_4B_RGB — reliable, fast, low VRAM.
- **For research/exploration**: Qwen2.5-VL-7B can work if you pre-filter images by size and skip those exceeding a resolution threshold.
- **To make Qwen viable on 32 GB**: add `max_pixels` constraint in the processor config to limit dynamic tiling, at the cost of detail on large images.
