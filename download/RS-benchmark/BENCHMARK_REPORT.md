## Key Findings

1. **EarthDial is rock-solid stable** on RS data — 0 errors, 100% completion. As an RS-specialized
   model, its fixed 448×448 resolution avoids the OOM issues that plague general-purpose VLMs.

2. **Qwen is faster but unreliable** — while Qwen has higher token throughput (25-32 tok/s vs
   4-17 tok/s for EarthDial), it fails on 15.3% of samples due to dynamic resolution OOM.

3. **EarthDial excels at RS-native tasks** — change detection (0.34s) and change captioning
   (0.60s) are much faster than Qwen (0.23s and 1.62s). EarthDial was designed for these tasks.

4. **EarthDial is more concise** — produces ~13,800 estimated tokens vs Qwen's ~44,582 tokens across
   all samples. For RS applications, concise, correct answers are often preferred.

5. **Trade-off**: Qwen's dynamic resolution gives richer descriptions when it works, but at the
   cost of reliability. EarthDial's fixed resolution is less detailed but never crashes.

## Recommendations

- For **production RS pipelines** where reliability matters: **EarthDial** is the clear winner.
- For **exploratory analysis** where detail matters more than reliability: Qwen can work, but
  needs input size filtering / resolution caps to avoid OOM.
- To make Qwen viable on 32GB GPUs, add image size validation before inference and skip images
  that would create >N visual tokens.
