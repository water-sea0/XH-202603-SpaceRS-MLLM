# SSH 反向端口转发

SSH 反向端口转发用于把本地电脑上的服务提供给远程服务器使用。
它与常见的本地转发 `-L` 方向相反，因此使用参数 `-R`。
基本格式如下：
```powershell
ssh -R 远程端口:本地主机:本地端口 用户名@远程服务器
```
例如，本地 Clash 的代理端口为 `7890`：
```powershell
ssh -N -T -R 17890:127.0.0.1:7890 root@服务器地址
```
此时远程服务器访问 `127.0.0.1:17890`，
流量会通过 SSH 隧道转发到本地电脑的 `127.0.0.1:7890`。
远程服务器可以这样测试：
```bash
curl -x http://127.0.0.1:17890 https://github.com
```
`-N` 表示不执行远程命令，只建立隧道。
`-T` 表示不分配交互终端，适合长期保持连接。
PowerShell 窗口必须保持运行，否则隧道会立即中断。
建议添加 `-o ExitOnForwardFailure=yes`，在转发失败时直接报错。
反向转发适合远程服务器复用本地代理、数据库或开发服务。
不要把远程监听地址设为公网可访问，除非已经配置认证和防火墙。
