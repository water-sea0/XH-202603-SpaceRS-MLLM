## 构建工具与编程文件
**这类文件是编译、构建或运行项目的“总指挥”，改名会导致构建失败。**
```
Makefile / makefile：Linux/Mac 下 make 命令的默认配置文件。它定义了编译规则，如果改名，就必须用 -f 参数指定。

Dockerfile：docker build 命令默认寻找的无扩展名文件，用于构建镜像。如果改名，构建时必须用 -f 参数指定。

CMakeLists.txt：cmake 命令的入口文件，定义了构建项目的规则，文件名是固定的。

package.json：Node.js 项目的核心文件，定义了项目依赖、脚本等，npm 或 yarn 依赖它。

requirements.txt / pyproject.toml：Python 项目的依赖管理文件，pip 等工具默认会寻找它们。

webpack.config.js / vite.config.js：前端构建工具的默认配置文件。
```
## 版本控制与忽略文件
**这类文件控制着代码的版本管理和哪些文件不应该被跟踪。**
```
.gitignore：告诉 Git 哪些文件或目录不需要纳入版本控制。它通常放在项目根目录，文件名固定，且以点开头，在 Linux 下是隐藏文件。

.gitmodules：当项目使用 Git 子模块时，这个文件定义了子模块的路径和源。

.dockerignore：类似于 .gitignore，告诉 Docker 在构建镜像时忽略哪些文件。
```
## 操作系统与系统文件
**这类文件通常关系到系统启动或核心功能，名字被内核或系统服务“硬编码”。**
```
/boot/vmlinuz-*：Linux 内核的镜像文件，文件名包含版本号，由引导程序（如 GRUB）直接读取，不能随意改名。

/etc/fstab：Linux 系统中，文件系统表的配置文件，定义了硬盘分区、挂载点等。

/etc/passwd、/etc/shadow：用户账户信息的核心文件，系统命令依赖它进行用户验证。

/boot/grub/grub.cfg：GRUB 引导器的配置文件，若改错名字，系统将无法正常启动。

initrd / initramfs：初始 RAM 磁盘，在 Linux 内核启动时被加载，包含必要的驱动和脚本。
```
## 环境与 Shell 配置
**这类文件在用户登录或开启新 Shell 时被自动执行，用于设置个性化环境。**
```
.bashrc、.bash_profile、.profile：Bash Shell 的配置文件。

.zshrc：Zsh Shell 的配置文件。

/etc/environment：系统级的环境变量配置文件。
```
## 项目文档与约定
**这类文件是项目“门面”，更改名字会破坏社区约定，影响他人对项目的理解。**

README.md / README：项目的说明文件。代码托管平台（如 GitHub）会在仓库首页自动渲染它。
```
LICENSE / LICENSE.md：项目的许可证文件。

CONTRIBUTING.md：为项目贡献代码的指南文件。

CODE_OF_CONDUCT.md：行为准则文件。
```
## Windows 系统保留名
**在 Windows 上，有一些名字是“神圣不可侵犯”的，因为它们是系统预留的设备名。**
```
CON、PRN、AUX、NUL

COM1-COM9、LPT1-LPT9

CLOCK$
```
