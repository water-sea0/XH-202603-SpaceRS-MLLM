# venv 与 conda 虚拟环境操作对比

## venv

**创建**：`python -m venv venv`

**激活**：`venv\Scripts\activate`（Windows）或 `source venv/bin/activate`（Linux/Mac）

**退出**：`deactivate`

**导出**：`pip freeze > requirements.txt`

**导入**：先激活新环境，再 `pip install -r requirements.txt`

**删除**：退出环境后直接删文件夹。

## conda

conda 环境可以建在**两个地方**、用**三种方式**使用。

### 创建

```
conda create -n 名字 python=3.10        ← 集中装在 conda 自己的 envs 文件夹里
conda create -p .\.conda_env python=3.10 ← 就地装在当前目录下
```

### 激活

```
conda activate 名字                      ← 集中式用名字
conda activate D:\项目\.conda_env        ← 就地式用路径
conda deactivate                         ← 退出
```

### 不激活直接用

```
conda run -n 名字 python 脚本.py         ← 集中式
conda run -p .\.conda_env python 脚本.py ← 就地式
```

> `conda run` 比直接指定 `python.exe` 路径更安全——它会正确设置环境变量，跑 gdal、CUDA 等依赖系统库的代码不会报 DLL 找不到。如果activate失败也可以尝试使用`conda run`。

### 导出与导入

```
conda env export -n 名字 > environment.yml          ← 导出

conda env create -n 名字 -f environment.yml          ← 导入（集中式）
conda env create -p .\.conda_env -f environment.yml  ← 导入（就地式）
```

### 删除

```
conda env remove -n 名字                ← 集中式
conda env remove -p .\.conda_env        ← 就地式
```
